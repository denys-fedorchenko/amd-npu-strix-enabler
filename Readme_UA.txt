# AMD Ryzen AI NPU Enabler (Ubuntu 26.04)

> [!CAUTION]
> **КРИТИЧНА ЗАЛЕЖНІСТЬ: PYTHON 3.14**
> Бінарні файли XRT (v2.23.0) зібрані спеціально під інтерфейс **Python 3.14**.
> Використання середовищ Conda з Python 3.13 або старішим призведе до помилки `ModuleNotFoundError` при спробі імпорту `pyxrt`.
>
> **Рішення**: Використовуйте системний `/usr/bin/python3` або створіть середовище з правильною версією: `conda create -n npu python=3.14`.

---

## 🖥️ Системні вимоги
* **Апаратне забезпечення**: Процесор AMD Ryzen з інтегрованим NPU (Zen 5 / Ryzen AI 7 350 / Krackan).
* **ОС**: Ubuntu 24.04 / 26.04 LTS (Ядро 6.8 - 7.0+).
* **Залежності**: Локальні архіви XRT v2.23.0 у папці `src/`.

---

## 🚀 Основна логіка та архітектура
Цей набір інструментів реалізує **Zero-Conflict SRE Architecture** для систем, де офіційна підтримка XRT ще не стабілізована в репозиторіях ОС.

### 1. Рівень сумісності (Dummy Package)
Щоб обійти блокування через версії системних бібліотек в Ubuntu 26.04, скрипт генерує пакет `xrt-base-dummy`:
* Реєструється як віртуальна залежність для задоволення вимог менеджера плагінів.
* Запобігає видаленню драйверів NPU під час виконання `apt upgrade`.

### 2. Пряме розгортання (Bypass Mode)
Замість ризику зламати системні `libc` або `boost`, файли XRT розгортаються методом прямої екстракції:
* Всі бінарні файли копіюються безпосередньо в `/opt/xilinx/xrt/`, що ізолює середовище NPU від системних оновлень.

### 3. Права доступу та ліміти пам'яті
* **Правила UDEV**: Надає доступ до `/dev/accel/accel0` з правами `0666` для роботи без sudo.
* **Ліміти пам'яті**: Встановлює `memlock unlimited`, що є критичним для коректного мапування пам'яті (mmap) під час ШІ-обчислень.

FAQ (Часті питання)
П: Чому під час налаштування з'явилося повідомлення "Autocomplete not enabled"? В: Це некритичне попередження оболонки щодо автодоповнення XRT. Інсталятор приглушує його для чистоти терміналу.

П: Чи збережеться драйвер після оновлення ядра? В: Так. Модуль amdxdna зареєстрований через DKMS, тому він автоматично перезбирається при оновленні Linux.

П: Як отримати доступ до NPU без sudo? В: Інсталятор автоматично налаштовує правила udev. Якщо доступ заборонено, переконайтеся, що ваш користувач доданий до групи render або video.
